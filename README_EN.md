# GHS ELF Memory Analysis Tool

A graphical tool for analyzing ELF files generated by the GHS compiler and generating Excel reports for ROM and RAM usage statistics.

## Features

- 📊 Parse ELF files generated by GHS compiler
- 📈 Automatically classify memory sections (ROM, RAM, Stack, Heap, etc.)
- 📑 Generate detailed Excel statistical reports
- 🖥️ User-friendly graphical interface
- 📦 Support custom memory region total capacity configuration
- 🎯 Quickly evaluate project ROM/RAM usage
- 🔍 Symbol table parsing, statistics for actual usage and remaining capacity of each section
- 📍 Automatically detect address gaps, display complete memory map layout
- 🌐 Multi-language support (Chinese/English)

## Usage

### Start GUI Tool

```bash
python main.py
```

### Operation Steps

1. **Select ELF File**: Click "Browse" button to select ELF file generated by GHS
2. **Specify Output Path**: Select save location for Excel report (auto-generated by default)
3. **Configure Total Capacity** (Optional):
   - Enter ROM total capacity (KB)
   - Enter RAM total capacity (KB)
   - If not entered, usage rate will not be displayed in report
4. **Start Analysis**: Click "Start Analysis" button
5. **View Report**: After analysis is complete, Excel report will be automatically saved

### Language Selection

- Click the language dropdown in the top-right corner
- Select "zh_CN" for Chinese or "en_US" for English
- Restart the application for the language change to take effect

## Excel Report Contents

The generated Excel file contains the following worksheets:

### 1. Memory Summary
- ROM (Flash/Code) usage
- RAM (Data/BSS) usage
- Stack and Heap usage
- Section size, actual usage, remaining capacity statistics

### 2. Detailed Sections
- Complete list of all memory sections
- Includes section name, address range, size, actual usage, remaining capacity, and type
- Automatically detects and displays address gaps

### 3. ROM Details
- Detailed breakdown of ROM-related sections
- Includes .text, .rodata and other code and read-only data sections
- Shows actual usage and remaining capacity for each section

### 4. RAM Details
- Detailed breakdown of RAM-related sections
- Includes .data, .bss and other initialized and uninitialized data sections
- Shows actual usage and remaining capacity for each section

## Memory Section Classification Rules

The tool automatically classifies sections in ELF files as:

- **ROM**: Code sections and read-only data (.text, .rodata, .init, etc.)
  - Sections with addresses starting with 0x00 are automatically classified as ROM
- **DATA**: Initialized data sections (.data, .sdata, etc.)
- **BSS**: Uninitialized data sections (.bss, .sbss, etc.)
- **RAM**: DATA + BSS
  - Sections with addresses starting with 0xFE are automatically classified as RAM
- **STACK**: Stack sections
- **HEAP**: Heap sections
- **CUSTOM**: Other custom sections

## FAQ

### Q: What ELF file formats does the tool support?
A: The tool uses the pyelftools library and supports standard ELF format files, with optimizations specifically for ELF files generated by the GHS compiler.

### Q: How to get ROM/RAM total capacity?
A: You can obtain it from the following sources:
- Check chip datasheet
- Check MEMORY definition in linker script (.ld or .dld file)
- If unsure, leave it blank, the tool will still display actual usage

### Q: How is actual usage calculated?
A: The tool parses the ELF file's symbol table and counts the total size of all function and data object symbols in each section. Uses interval merging algorithm to handle symbol overlap, ensuring accurate statistics.

### Q: What to do if analysis fails?
A: Please check:
- Whether ELF file is complete and not corrupted
- Whether file path contains special characters
- Whether there is enough disk space to save Excel report
- Whether output file is already opened by another program
